# DoG Defense Q&A

Это короткая шпаргалка под пересдачу. Ее цель не в том, чтобы выучить весь PHP, а в том, чтобы не зависнуть на базовых вопросах преподавателя и уметь быстро объяснить, что происходит в проекте.

## 1. Что такое `::` в PHP

`::` в PHP это **scope resolution operator**. Его используют, когда обращаются к:

- статическому методу класса;
- статическому свойству класса;
- константе класса;
- `parent::method()` или `self::method()`.

Примеры:

```php
Database::reset();
UserRepository::findById(1);
self::build();
parent::save();
```

Смысл:

- `->` используют для **объекта**;
- `::` используют для **класса** или статического контекста.

Пример разницы:

```php
$user->save();
User::findById(1);
```

Как отвечать устно:

> `::` это оператор доступа к статическим методам, свойствам или константам класса. Он не работает с обычным объектом, для объекта используется `->`.

Важно для вас: в текущей упрощенной версии проекта мы почти убрали такие вызовы из runtime-кода, чтобы код был ближе к лабораторкам и проще для изменения.

## 2. Можно ли ответить на любой вопрос

Честно: на **любой** вопрос ответить нельзя. Но можно очень сильно повысить шанс, что вы ответите на 80-90% вопросов, если знать:

- базовый синтаксис PHP;
- как работает `fetch()` и JSON;
- как работает сессия;
- как делаются prepared statements;
- где в проекте логин, генерация, экспорт, роли, CSV и SQLite.

Самая частая причина завала не в сложных темах, а в том, что студент не может объяснить:

- что делает конкретная строка;
- почему здесь именно PHP, JSON или `fetch`;
- чем отличается `GET` от `POST`;
- где хранится пользователь;
- как устроена база.

## 3. Что говорить, если не уверен

Никогда не молчать 20 секунд.

Лучше говорить так:

> Nu sunt 100% sigur pe formulare, dar aici ideea este...

или по-русски про себя сформулировать, потом сказать простыми словами:

> Здесь идея такая: браузер отправляет запрос, PHP его обрабатывает, потом данные либо берутся из SQLite, либо сохраняются туда.

То есть лучше дать правильную идею простыми словами, чем пытаться звучать “умно” и ошибиться.

## 4. Самые вероятные вопросы по проекту

### Что делает проект

> Это generator/manager de documente. Пользователь выбирает шаблон, источник данных, потом сервер генерирует HTML и PDF документ, сохраняет его в SQLite и даёт превью и экспорт.

### Почему без framework

> Потому что по условию дисциплины framework-urile Web nu sunt permise. Поэтому использованы plain PHP, SQLite, HTML, CSS и vanilla JavaScript.

### Где тут AJAX

> В `assets/js/app.js` используется `fetch()` для асинхронных запросов к `api.php`, без перезагрузки страницы.

### Где тут Web service

> `api.php` выступает как собственный HTTP API приложения. Он принимает `action`, читает JSON и возвращает JSON или файл.

### Где используется JSON

> JSON используется в запросах из browser в `api.php`, в ручном режиме генерации документа и в export `download_json`.

### Где используется CSV

> CSV используется в режиме import. Браузер читает файл через `FileReader`, потом сервер разбирает CSV с помощью `str_getcsv`.

### Почему SQLite

> Потому что это минимальная relational database, которую прямо рекомендуют в условии проекта. Она удобна для локального запуска и для учебного проекта.

### Где защита от SQL injection

> Все запросы с пользовательскими данными идут через `PDO::prepare()` и `execute()` с параметрами, а не через склейку SQL-строк.

### Где защита от XSS

> Значения, которые попадают в шаблон документа, проходят через `htmlspecialchars()`, поэтому HTML/JS из ввода не выполняется как код.

### Где роли

> Роль хранится в таблице `users`, а на сервере права проверяются в `require_user()`, `require_admin()` и `require_editor()`.

### Почему недостаточно скрыть кнопку на frontend

> Потому что безопасность должна проверяться на сервере. Даже если пользователь вручную вызовет `api.php`, `require_admin()` или `require_editor()` всё равно вернут 403.

### Где admin module

> В интерфейсе есть секция Admin, а на сервере есть actions `users`, `user_role`, `template_create`.

### Где генерируется PDF

> На сервере, в `lib/PdfBuilder.php`. После генерации HTML текст преобразуется в простой PDF и сохраняется в `exports/`.

### Почему PDF простой

> Потому что задача требует export în PDF, а не full publishing engine. Текущая реализация делает валидный минимальный PDF и показывает понимание формата и server-side export.

### Где хранится логин

> После успешного входа сервер выдаёт JWT token. Browser сохраняет token и отправляет его в `Authorization: Bearer ...`, а сервер по этому token определяет пользователя.

### Что такое JWT

> JWT înseamnă JSON Web Token. Это подписанный token, который содержит payload, например `sub`, `email`, `role`, `iat`, `exp`. Сервер проверяет подпись и срок жизни token-ului.

### Что делает `password_hash()`

> Хэширует пароль перед хранением в базе. Мы не храним пароль в открытом виде.

### Что делает `password_verify()`

> Сравнивает введённый пароль с хэшом из базы.

### Что делает `fetch()`

> Отправляет HTTP-запрос из браузера. У нас через `fetch()` frontend вызывает `api.php`.

### Чем отличается `GET` от `POST`

> `GET` обычно используют для чтения данных. `POST` используют для отправки или изменения данных. В проекте login/generate/user_role идут через `POST`.

### Что делает `json_decode()`

> Преобразует JSON-строку в PHP-массив или объект. У нас используется для чтения тела запроса и для чтения `data_json` из базы.

### Что делает `json_encode()`

> Преобразует PHP-массив в JSON-строку. У нас используется в API-ответах и при сохранении `data_json`.

## 5. Вопросы по PHP-синтаксису

### Что такое `<?php`

Это открывающий тег PHP-кода.

### Что такое `$variable`

В PHP переменные начинаются с `$`.

### Что такое ассоциативный массив

Это массив вида:

```php
[
    'name' => 'Ana',
    'role' => 'editor'
]
```

Доступ:

```php
$user['role']
```

### Что такое `require_once`

Подключает другой PHP-файл один раз. Если подключить повторно, второй раз код не выполнится.

### Что такое `if (...) { ... }`

Обычное условие.

### Что такое `foreach`

Цикл по массиву.

```php
foreach ($users as $user) {
    echo $user['email'];
}
```

### Что такое `return`

Возвращает значение из функции.

### Что делает `exit`

Сразу завершает выполнение скрипта.

## 6. Вопросы по JavaScript

### Что делает `document.getElementById`

Находит HTML-элемент по `id`.

### Что делает `addEventListener`

Привязывает обработчик события, например submit или click.

### Что делает `event.preventDefault()`

Останавливает стандартное поведение браузера. Например, чтобы форма не перезагружала страницу.

### Что делает `FormData`

Собирает значения полей формы.

### Что делает `FileReader`

Читает локальный файл в браузере. У нас так читается CSV перед отправкой на сервер.

### Что делает `innerHTML`

Вставляет HTML внутрь элемента.

## 7. Вопросы по SQL и SQLite

### Что такое primary key

Уникальный идентификатор записи в таблице.

### Что такое foreign key

Связь между таблицами. Например, документ хранит `template_id` и `created_by`.

### Что делает `SELECT`

Читает данные.

### Что делает `INSERT`

Добавляет новую запись.

### Что делает `UPDATE`

Изменяет существующую запись.

### Что делает `JOIN`

Объединяет данные из нескольких таблиц по связи между ними.

Пример из проекта:

- документы соединяются с шаблонами;
- документы соединяются с пользователями.

## 8. Что он может попросить изменить прямо на месте

### Поменять роль по умолчанию

Смотреть:

- `lib/Database.php`
- `lib/Repositories.php`

### Добавить новый template placeholder

Смотреть:

- `lib/TemplateEngine.php`

Например он может попросить:

- `{{faculty}}`
- `{{group}}`
- `{{year}}`

### Добавить ещё одну колонку в таблицу users или documents

Смотреть:

- `lib/Database.php`
- SQL schema
- соответствующие `INSERT` и `SELECT` в `lib/Repositories.php`

### Добавить новый export

Смотреть:

- `api.php`

Например ещё один `action`.

### Изменить порядок документов

Смотреть:

- SQL `ORDER BY` в `document_list_all()`

### Запретить editor создавать шаблон

Уже запрещено, но если он попросит поменять доступы:

- `require_admin()`
- `require_editor()`
- соответствующие `if ($action === ...)`

### Убрать random mode

Смотреть:

- `assets/js/app.js`
- `api.php`
- `lib/FakeData.php`

### Сделать новый тип документа

Нужно:

1. добавить шаблон в `db_initialize()`;
2. при желании добавить random data;
3. выбрать его в интерфейсе.

## 9. Как готовиться за ночь

Не учить всё подряд. Учить слоями:

1. Открыть `index.php` и понять структуру страницы.
2. Открыть `assets/js/app.js` и понять, как форма отправляет `fetch()`.
3. Открыть `api.php` и пройти все `action`.
4. Открыть `lib/Repositories.php` и понять SQL.
5. Открыть `lib/TemplateEngine.php` и понять placeholders.
6. Открыть `lib/Database.php` и понять таблицы.

Если знаете эти 6 файлов, уже будет очень тяжело вас поймать на базовых вопросах.

## 10. Самая короткая схема ответа на любой вопрос

Если не знаешь, начинай так:

1. Это происходит на frontend или backend?
2. Какие данные приходят на вход?
3. Что код делает с этими данными?
4. Что он возвращает или сохраняет?

Пример:

> На frontend форма собирает данные, потом через `fetch()` отправляет их в `api.php`. На backend PHP читает JSON, проверяет пользователя, обрабатывает шаблон, сохраняет документ в SQLite и возвращает preview и ссылки на export.

Это почти всегда лучше, чем молчание.

## 11. Минимум, который надо выучить наизусть

- `::` это доступ к статическому контексту класса.
- `->` это доступ к объекту.
- `fetch()` делает HTTP-запрос.
- JWT token хранит данные аутентификации на клиенте и отправляется в `Authorization`.
- `PDO prepare + execute` защищают от SQL injection.
- `htmlspecialchars()` защищает от XSS при выводе.
- `json_encode()` кодирует массив в JSON.
- `json_decode()` читает JSON.
- `str_getcsv()` разбирает CSV.
- `require_once` подключает файл один раз.
- `POST` для отправки данных, `GET` для чтения.

Если это вы знаете спокойно, уже не будет ощущения, что вас можно завалить одной строчкой кода.
