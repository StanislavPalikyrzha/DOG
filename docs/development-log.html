Development log

Work log

This page records the main implementation stages used by the team while building the DoG project.

Timeline

  1. Project skeleton, licensing and local run instructions were prepared.
  2. The PHP bootstrap layer, session handling and SQLite initialization were added.
  3. Authentication, role checks and user persistence were implemented.
  4. Template, document, import and audit repositories were separated.
  5. The templating engine was extended with placeholders, conditional blocks and date helpers.
  6. Document generation was connected to JSON, CSV and realistic random data sources.
  7. HTML, JSON and PDF exports were exposed through application endpoints.
  8. The responsive dashboard, generator panel, admin panel and resource links were assembled.
  9. Scholarly HTML, architecture, design notes, data provenance and progress documents were added.
  10. Manual checks covered login, generation, import, export, admin access and viewer restrictions.
