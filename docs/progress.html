Development timeline

Intermediate stages

The project evolved from a document generator concept toward a modular submission-ready web application with admin tooling and import/export flows.

Stage 1

Defined the topic, core entities and the minimal server-side routes required for templates, documents, authentication and admin operations.

Stage 2

Implemented SQLite persistence, template rendering, PDF export and import pipelines for JSON and CSV sources.

Stage 3

Connected the browser interface to the PHP services through asynchronous fetch requests, then added audit logging, admin role management and submission documentation.
