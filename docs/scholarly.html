DoG: Document Web Generator

DoG is a modular web system for generating and managing HTML and PDF documents starting from predefined templates. The project focuses on server-side Web technologies, asynchronous client-server communication and open data formats.

Functional requirements

  • Authenticate users and enforce role-based access.
  • Generate documents from predefined templates using JSON, random data or CSV input.
  • Store generated documents and expose HTML and PDF exports.
  • Provide an administration module for users, templates and audit history.
  • Use SQLite for persistence and fetch-based AJAX for browser communication.

User interaction

The application exposes a responsive interface with panels for login, dashboard metrics, document generation, preview, document storage and administration. All write operations use asynchronous calls to api.php. The user can choose a template, select an input mode, submit data and immediately preview the server-rendered result.

System requirements

  • PHP 8 and PDO SQLite support.
  • Modern browser with Fetch API support.
  • No client-side or server-side frameworks.
  • Open formats for import/export: CSV, JSON, HTML and PDF.

Security and quality requirements

Database writes use prepared statements, generated values are escaped on the server, script tags are stripped from rendered previews, and role checks protect privileged routes. The codebase is modularized into repositories, authentication, templating and export helpers.
