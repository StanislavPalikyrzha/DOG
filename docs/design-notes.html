UI design rationale

Design notes

The interface avoids framework-generated dashboard patterns and instead uses a calmer academic workspace aesthetic: grid-paper background, serif typography, bordered panels and direct visual grouping of tasks.

Motivations

  • Keep the application understandable for a course project built without frontend frameworks.
  • Separate login, generation, storage and admin flows into explicit panels.
  • Preserve readability for longer template previews and form-heavy interactions.
