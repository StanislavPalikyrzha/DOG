Data model and provenance

Data sources

The project uses open and controllable sources only: SQLite seed data, manual JSON payloads, CSV imports prepared by the team, and realistic random data generated server-side for demos.

Persistent entities

  • users: login credentials, display names and roles.
  • templates: reusable HTML and CSS definitions for generated documents.
  • documents: generated output, source data and export metadata.
  • import_jobs: CSV import traceability.
  • audit_log: actions relevant to administration and generation.

Provenance rules

  • Manual mode receives JSON objects entered in the browser.
  • CSV mode receives open-format comma-separated values.
  • Random mode uses a small in-project generator for realistic but synthetic examples.
  • No proprietary SDKs or framework wrappers are used.
